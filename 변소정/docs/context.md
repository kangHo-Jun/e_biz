# 프로젝트 컨텍스트 (Project Context)

이 문서는 새로운 세션에서 에이전시 AI가 프로젝트의 현재 상태를 즉시 파악하고 일관된 기술적 결정을 내릴 수 있도록 돕기 위해 작성되었습니다.

## 1. 프로젝트 개요
- **프로젝트 명**: 쇼핑몰 배차노트 자동 파싱 시스템
- **목적**: 슬랙(Slack)에서 구글 시트 A열로 결제 안내 메시지를 붙여넣을 때, 날짜, 주문번호, 금액, 품목명 등을 자동으로 파싱하여 지정된 열(E, F, J, L, M)에 입력하는 자동화 시스템.
- **주요 기술**: Google Apps Script (GAS), Clasp (Deployment)

## 2. 현재 상태 및 완료된 기능
- **트리거 시스템**: GCP 권한 이슈를 피하기 위해 `onEdit` 단순 트리거를 사용하여 즉각적인 파싱 실행.
- **다중 행 지원**: 여러 행을 한꺼번에 붙여넣어도 전체 행을 순차적으로 파싱함.
- **데이터 추출 (Code.js)**:
    - **날짜(E) / 요일(F)**: 정규식 기반 추출.
    - **금액(J)**: 콤마 제거 및 숫자 형식화.
    - **주문번호(L)**: 앞자리 `0`이 보존되도록 텍스트 형식(`@`) 설정 후 입력.
    - **품목명(M)**: 이름/이모지 제거 로직 완료. 빈 줄이 섞여 있어도 첫 번째 하이픈(`-`) 이후의 내용을 정확히 추출하는 Robust 로직 적용.
- **배포 환경**: `clasp`를 통해 로컬에서 구글 스크립트로 직접 푸시 가능하도록 세팅됨.

## 3. 기술적 제약 및 주의 사항 (반드시 준수)
- **단순 트리거 사용**: `SpreadsheetApp.getActiveSpreadsheet()` 등의 권한이 필요한 API 대신 단순 트리거에서 허용되는 범위 내의 API 사용 (또는 권한 부여 확인 필수).
- **품목명 추출 규칙**: 품목명은 반드시 `[결제완료 안내]` 헤더 다음의 실제 내용 줄에서 첫 번째 `-` 이후를 가져와야 함 (이름/이모지 제거 목적).
- **주문번호 형식**: 주문번호 입력 시 `.setNumberFormat('@')`을 사용하여 문자열 형식을 유지해야 함 (숫자 변환 시 앞자리 0 손실 방지).
- **메시지 정규화**: `split('\n').map(l => l.trim()).filter(l => l.length > 0)` 패턴을 사용하여 슬랙 메시지의 불규칙한 빈 줄 문제를 해결해야 함.

## 4. 남은 작업 및 향후 계획
- **예외 케이스 모니터링**: 새로운 형식의 메시지가 들어올 경우 추출 규칙 추가 보완.
- **UI/UX 개선**: 시트 내에서 파싱 상태를 알 수 있는 상태 메시지(예: O/X 표시) 추가 고려 가능.
