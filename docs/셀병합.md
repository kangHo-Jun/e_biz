> **문서 유형**: 개발 지시서 (코딩 에이전시 감시용)
> **작성일**: 2026-02-26

## 1. 변경 개요

### 1.1 현재 구조 (AS~AV, 4개 열)

| 열 | 컬럼번호 | 내용 | 예시 |
|----|----------|------|------|
| AS | 45 | 규격 | `110바` |
| AT | 46 | 너비 | `880` |
| AU | 47 | 구분자 | `*` |
| AV | 48 | 높이 | `2090` |

### 1.2 변경 후 구조 (AS~AT, 2개 열)

| 열 | 컬럼번호 | 내용 | 예시 |
|----|----------|------|------|
| AS | 45 | 규격 | `110바` |
| AT | 46 | 너비*높이 (병합) | `880*2090` |

> **AU, AV 열은 삭제된다.** AT 이후 모든 열이 **2칸 앞으로 시프트**된다.

### 1.3 적용 행 범위

- 문틀(Frame): **AT12:AV20** → AT12 하나로 병합
- 도어(Door): **AT22:AV34** → AT22 하나로 병합

## 3. 파싱 유틸 함수 (신규 추가)

병합된 AT 값에서 너비/높이를 분리하는 공통 함수를 **1개** 만든다.
모든 수정 지점에서 이 함수를 호출한다.

```js
/**
 * 병합된 AT셀 값을 파싱하여 너비/높이를 분리한다.
 * @param {*} atValue - AT셀 값 (예: "880*2090", 880, "", null)
 * @returns {{ width: number, height: number, raw: string }}
 *
 * 예시:
 *   parseAT("880*2090") → { width: 880, height: 2090, raw: "880*2090" }
 *   parseAT("880")      → { width: 880, height: 0,    raw: "880" }
 *   parseAT("")          → { width: 0,   height: 0,    raw: "" }
 */
function parseAT(atValue) {
  if (!atValue) return { width: 0, height: 0, raw: "" };
  var str = atValue.toString().trim();
  var parts = str.split("*");
  return {
    width:  Number(parts[0]) || 0,
    height: Number(parts[parts.length > 1 ? parts.length - 1 : -1]) || 0,
    raw:    str
  };
}
```

## 7. 주의사항

1. **시트 먼저 수정 금지**: 코드를 먼저 수정하고 `clasp push`한 뒤, 시트에서 AU/AV 열을 삭제할 것. 순서가 뒤바뀌면 기존 onEdit가 오류를 발생시킴.
2. **백업**: 수정 전 `door_v4.gs`를 `door_v4_backup.gs`로 복사해 둘 것.
3. **parseAT 방어 코드**: `*`가 없는 숫자만 입력된 경우(`880`)도 처리 가능해야 함.
4. **열 문자 vs 열 번호**: 코드에서 `"BC2"` 같은 A1 표기와 `55` 같은 열 번호가 혼용됨. 둘 다 빠짐없이 수정할 것.
5. **테스트 시트 참조**: `테스트` 시트의 AG/AH 열 참조는 영림발주서 시트의 열 삭제와 무관하므로 변경 불필요.
