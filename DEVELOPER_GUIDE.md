# 📘 영림발주서 스크립트 개발 문서 (Developer Guide)

이 문서는 `door.gs` 스크립트의 **내부 구조, 핵심 로직, 데이터 흐름**을 설명합니다. 유지보수 및 기능 확장을 위한 기술 문서입니다.

---

## 1. 📂 프로젝트 구조

### 메인 파일
*   **`door.gs`**: 현재 운영 중인 최신 스크립트입니다. 모든 기능(단가 계산, 코드 생성, 트리거)이 여기에 통합되어 있습니다.

### 레거시/백업
*   `ver1.gs`, `door_v1.gs`: 이전 버전의 코드들로, 현재는 참조용으로만 존재합니다. 실 사용 시 `door.gs`만 수정해야 합니다.

---

## 2. ⚙️ 핵심 함수 명세 (API)

### 2.1 이벤트 트리거 (`onEdit`)
사용자 입력을 실시간으로 감지하여 자동완성을 수행합니다.

*   **함수명**: `onEdit(e)`
*   **파일 위치**: `door.gs` (Line 1630~)
*   **진입 조건**: 시트명이 "영림발주서"인 경우
*   **주요 로직**:
    1.  **체크박스 감지 (`BC2`)**: `true`가 되면 `생성_품목코드_문틀()` 실행
    2.  **색상 자동완성 (`AW` 열 수정)**:
        *   `Rows 12~35`: `COLOR_MAP` 속성 참조하여 **AX(코드)** 자동 입력
    3.  **가스켓/상태 자동완성 (`AW` 열 수정)**:
        *   `Rows 12~20`: `GASKET_COLOR_MAP` 속성 참조하여 **BA** 열 입력 ("단종", "5000" 등)
    4.  **문짝 방향 자동완성 (`AV` 열 수정)**:
        *   `Rows 22~34`: `AV >= 2166` 조건 만족 시 '테스트' 시트(AG/AH) 검색 후 **AQ(Y/N)** 입력

### 2.2 단가 계산 (`계산_영림발주서_가격_내부`)
복잡한 가격 정책을 적용하여 A열에 최종 단가를 출력합니다.

*   **함수명**: `계산_영림발주서_가격_내부()`
*   **수행 단계**:
    1.  **초기화**: `영림문틀단가표` 시트(C6:F500)와 `테스트` 시트(추가금 데이터)를 로드
    2.  **행 순회 (12~34행)**:
    3.  **Base Price 계산**:
        *   표준 규격: 단가표 매칭 가격
        *   비표준/실패 시: 기존 A열 수동 입력값 유지 (**Fallback Logic**)
    4.  **Option Price 가산**:
        *   **문틀 (12~20행)**: `BA` 열 값이 있으면 +5,500원
        *   **문짝 (22~34행)**: `AW`(색상)/`AX`(코드)에 따른 추가금 가산 (V~Z열, AD~AF열 참조)
    5.  **메모 마킹**: 계산 완료된 셀에 메모(Note)를 남겨 중복 계산 방지

### 2.3 품목 코드 생성 (`생성_품목코드_문틀`)
발주처(영림) 양식에 맞는 품목 코드를 생성합니다.

*   **함수명**: `생성_품목코드_문틀_내부()`
*   **구분 로직**:
    *   **문틀 (Rows 12~20)**: `구분_품목타입` 함수가 'FRAME'으로 판정 또는 행 번호 기준
        *   조합: `플래그코드(AP)` + `규격(AS+AT+AV)` + `색상(AX)`
    *   **문짝 (Rows 22~34)**: `구분_품목타입` 함수가 'DOOR'로 판정 또는 행 번호 기준
        *   조합: `모델코드(AP)` + `규격(AS+AT+AV+AQ)` + `색상(AX)`

## 3. 💾 데이터 구조 및 저장소

스크립트 속성(`ScriptProperties`)을 사용하여 대용량 매핑 데이터를 캐싱합니다.

### 3.1 `COLOR_MAP`
*   **출처**: `테스트` 시트 **V1:Z300**
*   **저장 형식**: JSON String
*   **구조**: `{"색상명": "색상코드", "코드": "색상명", ...}`
*   **업데이트 함수**: `updateColorCodeMap()` (파싱 로직 개선됨)

### 3.2 `GASKET_COLOR_MAP`
*   **출처**: `테스트` 시트 **M~U열**
*   **구조**:
    ```json
    {
      "M": { "제품명": { "value": "캐시미어", "isDiscontinued": false, ... } },
      "P": { ... },
      "S": { ... }
    }
    ```

## 4. 🛠️ 유지보수 포인트

### 4.1 '테스트' 시트 참조 범위 수정 시
데이터가 늘어나 참조 범위를 변경해야 한다면 다음 함수들을 수정하세요.
*   `updateColorCodeMap`: `testSheet.getRange("V1:Z300")` 부분
*   `onEdit`: AG/AH열 검색 범위 (`AG1:AH300`)

### 4.2 가격 정책 변경 시
*   **기수수료(5,500원) 변경**: `계산_영림발주서_가격_내부` 함수 내 `5500` 숫자 수정
*   **추가금 로직 변경**: `테스트` 시트 V3 헤더값 또는 코드 내 매칭 로직 확인

### 4.3 디버깅
*   `debugColorMapCheck()`: 색상 매핑 데이터가 제대로 로드되었는지 확인하는 도구입니다. 문제 발생 시 가장 먼저 실행해 보세요.
