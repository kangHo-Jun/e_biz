# 🚪 영림발주서 자동화 스크립트 (door.gs)

이 스크립트는 구글 스프레드시트의 '영림발주서' 시트에서 **단가 계산**과 **품목 코드 생성**을 자동화합니다.

---

## 🏗️ 품목 코드 생성 로직 (상세 가이드)

코드 생성은 **[확장 프로그램] > [Apps Script]** 메뉴 또는 **BC2 체크박스**를 통해 실행됩니다.

### 1. 문틀 (Rows 12 ~ 20)
문틀은 **AP열(품명)**에 포함된 키워드를 분석하여 코드를 조합합니다.

*   **참조 데이터**: `AP(품명)`, `AW(색상명)`, `AX(색상코드)`
*   **생성 규칙**:
    1.  **브랜드**: '영림' (기본값)
    2.  **타입 코드 (AP열 분석)**:
        *   `발포` → **YF**
        *   `방염` → **YF** (방염 표시 추가)
        *   `알루미늄` → **YA**
    3.  **규격 코드**: AP열에 포함된 숫자 (예: `110`, `140`, `155`, `175` 등)
    4.  **세부 옵션**:
        *   `가스켓` → **G**
        *   `민자` → **M**
    5.  **색상**: AX열의 색상 코드 사용

> **예시**: `영림 발포 110 가스켓` + `중백색(PW01)` → 코드: **YF-110G-PW01**

<br>

### 2. 문짝 (Rows 22 ~ 34)
문짝은 **모델명**과 **규격(치수)**, **색상**을 조합하여 코드를 생성합니다.

*   **참조 데이터**:
    *   `AP(품명)`: 모델명 추출 (예: TA-04, YA-01 등)
    *   `AS`, `AT`, `AV`: 가로, 세로 등 치수
    *   `AQ`: 식기/타공 여부 (Y/N/방향)
    *   `AX`: 색상 코드
*   **생성 규칙**:
    1.  **모델 코드 (AP열)**:
        *   영문+숫자 패턴 추출 (예: `TA-04` → **TA04**)
        *   `민무늬` → **MD**
        *   `탈공` → **탈**
    2.  **규격 코드 (AS+AT+AV+AQ)**:
        *   `AS(숫자)` + `AT(숫자)` + `AV(숫자)` + `AQ(방향)` 순서로 연결
        *   예: `800` + `2040` + `N` → **8002040N**
    3.  **색상 코드**: AX열 값 그대로 사용

> **예시**: `TA-04` + `800 x 2040` + `좌(N)` + `중백색(PW01)` → 코드: **TA048002040NPW01**

## ⚡ 자동 입력 기능 (실시간)

사용자가 데이터를 입력하는 순간 자동으로 동작하는 편의 기능입니다.

### 1. 색상 자동완성 (AX열)
*   **범위**: 12행 ~ 35행 (전체)
*   **동작**: **AW열(색상명)** 입력 시 → **AX열(코드)** 자동 입력
*   **데이터 출처**: `테스트` 시트 **V1:Z300** 영역 (형식: "영림116 (중백색)")

### 2. 가스켓/상태 자동입력 (BA열)
*   **범위**: 12행 ~ 20행 (문틀 구간)
*   **동작**: **AW열(색상명)** 입력 시 → **BA열**에 정보 입력
*   **내용**:
    *   `단종`, `단종예정` 상태인 경우 텍스트 입력
    *   가스켓 색상 정보가 있는 경우 색상 입력

### 3. 문짝 방향 자동입력 (AQ열)
*   **범위**: 22행 ~ 34행 (문짝 구간)
*   **실행 조건**: **AV열(세로)** 값이 **2166 이상**일 때만 동작 (AV ≤ 2165면 빈칸)
*   **상세 로직**:
    1.  **AW열(색상명)** 또는 **AX열(색상코드)** 값이 기준 정보에 있는지 확인
    2.  '테스트' 시트의 **AH열(34열)**에 포함된 단어면 → **"Y"**
    3.  '테스트' 시트의 **AG열(33열)**에 포함된 단어면 → **"N"**
    4.  둘 다 해당 없으면 빈칸 유지

## 💰 단가 계산

*   **대상**: A열
*   **실행**: 메뉴의 `단가계산` 버튼 클릭 시

### 1. 문틀 (12~20행)
*   **기본**: 규격과 품목에 따른 표준 단가 적용
*   **가스켓 옵션**: BA열에 값이 있으면 **+5,500원** 추가 (단, "없음", "단종", "단종예정" 제외)

### 2. 문짝 (22~34행) 가격 계산 상세 로직
문짝 가격은 매우 유동적이므로 다음의 **우선순위 로직**을 따릅니다.

1.  **기본 단가 결정**:
    *   **자동 계산**: AP(품명)가 등록된 표준 규격이면 단가표에서 가격을 가져옵니다.
    *   **수동 유지**: 자동 계산에 실패하면, **현재 A열에 입력된 값**을 기본 단가로 사용합니다. (사용자가 수동 입력 가능)

2.  **추가 금액 합산** (조건부 자동 추가):
    *   **색상 추가금 (Part 1)**: `AW열(색상명)`이 '테스트' 시트의 특정 그룹(V~Z열)에 속하면 해당 그룹의 추가금을 더합니다.
    *   **키워드 추가금 (Part 2)**: `AX열` 또는 `AW열` 값이 '테스트' 시트의 키워드 목록(AD:AF)에 있으면 해당 금액을 더합니다.

3.  **제외 조건**:
    *   문틀과 달리 **BA열(가스켓)** 값에 따른 5,500원 추가는 **적용되지 않습니다.**
    *   이미 계산 완료된 셀(✅ 메모 표시)은 재계산하지 않습니다.

## ⚙️ 설정 가이드

### '테스트' 시트 데이터 매핑
스크립트는 `테스트` 시트의 데이터를 참조하여 동작합니다.

| 데이터 종류 | 범위 | 설명 |
|---|---|---|
| **품목 정보** | M1:U300 | 제품명, 가스켓색상, 단종정보 |
| **색상 매핑** | V1:Z300 | "코드 (색상명)" 형식의 데이터 파싱 |
| **AQ 자동완성** | AG1:AH300 | 문짝 방향(Y/N) 판단용 키워드 |

### 데이터 업데이트
`테스트` 시트 내용을 수정했다면 반드시 **[메뉴] > 데이터 업데이트** 관련 함수를 실행해야 반영됩니다.
